## Not bugs — intentional scaffolding

### Patient tabs (Episodes, Marcações, Documents, Faturação) — non-functional
The tab nav in `/patients/[id]` renders all tabs as static `<span>` elements with no click handlers. This is **intentional** — the source code has an explicit comment: *"related streams (agenda/clinical/invoicing) populate these panels later"*. The tabs are UI placeholders pending the patient portal and invoicing UI waves. Not a bug.

### No PDF download button on signed clinical record
The PDF generation library (`lib/clinical/report/`) is fully built in PR #122, but no download button or UI route has been wired yet. The engine exists, the front-end trigger is a follow-up task. Not a bug — same pattern as InvoiceXpress (backend done, UI pending).
