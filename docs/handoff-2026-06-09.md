## Recurring patterns / gotchas

### Lockfile conflicts (happens on EVERY portal PR)
Every PR that touches i18n or adds a new app hits pnpm-lock.yaml conflicts. Resolution:
```bash
git pull origin main
git checkout --ours apps/portal/lib/api/client.ts  # keep our version
pnpm install --no-frozen-lockfile
pnpm install --frozen-lockfile
git add pnpm-lock.yaml
git commit -m "chore: resolve merge conflict, regenerate lockfile"
git push
```

### `client.ts` is the single source of truth
ALL portal API types and functions live in `apps/portal/lib/api/client.ts`. Never create separate files like `lib/api/profile.ts` — they can't reference `apiBase()` and `apiHeaders()` which are private to client.ts. Caused TS2304 errors previously.

### Test assertions on unauthorized()
Ivan's `unauthorized()` helper (apps/api/lib/appointments/http.ts) returns `{ error: "unauthorized", message: "..." }`. Tests must assert `body.error === "unauthorized"` not deep-equal `{ error: "unauthorized" }`.

### Branch protection
Main is protected — all changes go through PRs. Never push directly to main.

### CI workflow versions
Check `.github/workflows/ci.yml` and `.github/workflows/e2e.yml` separately — they may use different pnpm versions.
