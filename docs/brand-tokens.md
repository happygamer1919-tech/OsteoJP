# Brand Tokens: OsteoJP Platform

> Single source of truth for the OsteoJP visual identity and the UI redesign.
> All product UI, marketing assets, and printed documents must reference these
> values. The Tailwind block in section 7 is the implementation contract for
> `packages/ui`.
>
> Companion doc: [`brand-voice.md`](./brand-voice.md) (tone and copy rules).
> Brand tone in UI: serious, precise, not warm. No emoji in product UI.

## 2. Typography

### 2.1 Font pairing

**Inter** (UI, body, forms, data) paired with **Source Serif 4** (long-form
print output: clinical reports, declarations, document headers).

Both are on Google Fonts with full **Latin Extended** support, covering all
pt-PT diacritics (ã, õ, á, à, â, é, ê, í, ó, ô, ú, ç). Load via Next.js
`next/font/google` with `subsets: ['latin', 'latin-ext']`.

| Role | Family | Fallback stack |
|---|---|---|
| UI / body | Inter | `'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui, sans-serif` |
| Documents / print | Source Serif 4 | `'Source Serif 4', Georgia, 'Times New Roman', serif` |

Rationale: Inter is neutral, clinical, excellent at small sizes (dense forms
and tables). Source Serif 4 gives printed reports and declarations the formal
register the brand requires without introducing a display face. Inter is
already the platform standard; this doc does not change it.

### 2.2 Type scale

Base 16px, modular ratio ~1.25. Tailwind-style names.

| Token | Size | Line height | Default weight | Use |
|---|---|---|---|---|
| `xs` | 12px / 0.75rem | 16px | 500 | Labels, badges, tags |
| `sm` | 14px / 0.875rem | 20px | 400 | Dense UI, table cells, helper text |
| `base` | 16px / 1rem | 24px | 400 | Body text |
| `lg` | 18px / 1.125rem | 26px | 500 | Minor headings, form group labels |
| `xl` | 20px / 1.25rem | 28px | 600 | Subsection titles, card headers |
| `2xl` | 24px / 1.5rem | 32px | 600 | Section titles |
| `3xl` | 32px / 2rem | 40px | 600 | Page titles |
| `4xl` | 40px / 2.5rem | 48px | 600 | Marketing hero, large dashboards (rare in app) |

### 2.3 Weights

- **400** body text (default)
- **500** labels, captions, slight emphasis
- **600** headings, button labels, strong emphasis
- **700** reserved; avoid, use 600 by default

## 4. Spacing scale

4px base unit. All padding, margin, and gap values snap to this scale. This
matches Tailwind's default 4px grid; the config pins the named steps explicitly.

| Token | Value | Use |
|---|---|---|
| `1` | 4px | Hairline gaps, icon-to-text inside compact components |
| `2` | 8px | Tight grouping (form label to input, badge padding) |
| `3` | 12px | Default component internal padding (buttons, inputs) |
| `4` | 16px | Standard gap between related elements |
| `6` | 24px | Section internal spacing, card padding |
| `8` | 32px | Spacing between distinct sections within a screen |
| `12` | 48px | Major section separation, page-level vertical rhythm |
| `16` | 64px | Top-level page padding on desktop, hero spacing |

Never invent off-scale values like `10px` or `18px`.

## 5. Shadow scale

Cool-tinted shadows derived from `neutral-900` (`rgb(26 39 51)`), never pure
black. Low opacity: elevation should read as subtle in a clinical UI.

| Token | Value | Use |
|---|---|---|
| `shadow-xs` | `0 1px 2px rgb(26 39 51 / 0.05)` | Inputs, subtle lift |
| `shadow-sm` | `0 1px 3px rgb(26 39 51 / 0.08), 0 1px 2px rgb(26 39 51 / 0.04)` | Cards at rest |
| `shadow-md` | `0 4px 8px -2px rgb(26 39 51 / 0.10), 0 2px 4px -2px rgb(26 39 51 / 0.06)` | Dropdowns, popovers, hover lift |
| `shadow-lg` | `0 12px 24px -4px rgb(26 39 51 / 0.12), 0 4px 8px -4px rgb(26 39 51 / 0.06)` | Modals, drawers |
| `shadow-overlay` | `0 24px 48px -12px rgb(26 39 51 / 0.18)` | Full-screen overlays, command palette |

## 6. Heritage theme (tenant-scoped decorative layer)

> **Status: pending JP sign-off for patient-facing surfaces.** Do not ship on
> any patient-facing surface until signed off (see QUESTIONS.md Q6).

A tenant-scoped decorative layer combining **Moldovan embroidery geometry**
(diamond and stepped-cross lattices) with **Portuguese azulejo motifs**
(four-fold rotational tile patterns). Purely ornamental: it carries no meaning,
no data, and no interaction.

### Rules (non-negotiable)

1. **Always recolored to the brand palette.** Motifs use only:
   `primary-200`/`primary-300`, `accent-1-200`, `accent-2-200`, `neutral-200`,
   at 100% tint or lighter. **Never** traditional folk red/black, and never any
   hex outside this document.
2. **Allowed surfaces only:** auth screens, empty states, loading states, and
   dividers (thin pattern strips).
3. **Never on data-dense screens:** agenda, patient lists, clinical record
   editor, invoicing, dashboards, tables, forms. No exceptions.
4. **Tenant-scoped:** the theme is a per-tenant setting. The **default tenant
   theme is neutral (no motifs)**. A tenant must explicitly opt in.
5. Motifs are background-layer only (CSS background or inline SVG behind
   content), never interleaved with interactive elements, and must pass
   contrast checks for any text rendered above them.

### Implementation sketch

A `heritage` flag in tenant settings selects between `neutral` (default) and
`heritage`. Pattern assets live in `packages/ui` as recolorable SVGs taking
`currentColor` or CSS variables, so the palette rule is enforced structurally
rather than by convention.
