## 🐛 Bug Log

### BUG-04 — Dashboard has no navigation
**Severity:** 🔴 Critical  
**Route:** `/dashboard`  
**Description:** After login, the dashboard shows only tenant ID, role, patient count, and a "Sair" button. There is no sidebar, no top nav, no links to any other section of the app. The user is completely stranded — every other route is only reachable by typing the URL directly. This must be fixed before any demo.  
**Expected:** A navigation menu linking to Agenda, Pacientes, Fichas Clínicas, Administração.

---

### BUG-05 — "+ Novo Episódio" button does nothing
**Severity:** 🔴 Critical  
**Route:** Patient profile (e.g. `/patients/:id`)  
**Description:** Clicking "+ Novo Episódio" on a patient profile produces no response — no navigation, no modal, no error in the UI. The core clinical workflow (create episode → fill clinical form → save) is entirely broken from this entry point.  
**Expected:** Should navigate to `/clinical/new?patient=:id` or open an episode creation modal with template selection.

---

### BUG-06 — "Nova Ficha" button on `/clinical` does nothing
**Severity:** 🔴 Critical  
**Route:** `/clinical`  
**Description:** Clicking "Nova Ficha" in the clinical records list produces no response — no navigation, no modal, no error. Same symptom as BUG-05.  
**Expected:** Should navigate to a new clinical record form (requiring patient + template selection).

---

### BUG-07 — "Sexo" field is a plain text input
**Severity:** 🟡 Medium  
**Route:** `/patients/new`  
**Description:** The "Sexo" field renders as a plain `<input type="text">` with no options. It should be a `<select>` dropdown.  
**Expected values:** Masculino / Feminino / Outro / Não especificado  
**Note:** Also relates to open question in `docs/i18n-copy-review.md` (item 6 — confirm "Biological sex" as EN label).
