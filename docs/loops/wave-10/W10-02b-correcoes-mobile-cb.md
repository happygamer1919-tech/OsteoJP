### Defect 1 - Bodychart marker list collapses to one-character-per-line on mobile
The screenshot (evidence) shows the ficha editor on a phone: rows reading `[anterior]` + a wrapped label + `Remover` where the label text wraps to ONE CHARACTER PER LINE vertically (e.g. "D / is / f / u / n / c / a / o ..."). The offender is the **Bodychart marker list**, not a generic table:
- Component `apps/web/app/clinical/[id]/BodyChart.tsx` (`BodyChart` at `:194`), rendered from `RecordForm.tsx:346-355` (the `"bodychart"` widget case, wrapped in a `<Card>`).
- The marker rows are a flex `<ul>/<li>`, NOT a `<table>`: `<ul className="space-y-1 text-sm">` (`BodyChart.tsx:413`); each row `<li className="flex min-w-0 flex-wrap items-center gap-2">` (`:420`); the `[view]` chip `[{m.view}]` (`:421`, renders `[anterior]`/`[posterior]`); the label `<span className="min-w-0 flex-1 break-words">{displayLabel(m)}</span>` (`:422`); the Remover button `clinical.bodychartRemove` (`:448-457`).
- **Root cause to confirm:** the canvas + list sit in `<div className="flex items-start gap-3">` (`:317`) with a FIXED-width, non-shrinking canvas `className="relative h-80 w-56 shrink-0 ..."` (`:327`, `w-56` = 224px). On a ~360px phone the fixed 224px canvas plus gaps starve the label column, so `flex-1 break-words` breaks at the only width it has (a character or two), producing the vertical letter-wrap. The label already carries `min-w-0 break-words`; the problem is the SIDE-BY-SIDE layout on narrow viewports, not the label class.
- **Fix intent:** on narrow viewports STACK the canvas and the marker list vertically (e.g. a responsive `flex-col` -> `sm:flex-row`, or a responsive grid) so the marker list gets the full row width; give the marker list / label column a sane `min-width` so labels wrap at WORD boundaries, never per character; keep each row a horizontal `[view]` chip + label (taking remaining width) + Remover. Confirm no OTHER array/list render in the ficha editor has the same per-character wrap (the legend list at `:394-405` uses the same idiom - check it too).
- **LOCK (W5-25, do NOT touch):** the bodychart plots nine `marker_type` values by SHAPE (authoritative) + colour, with an always-visible legend and the frozen `osteopathy-v3.json` enum; the stored marker `{ marker_type, x, y, view }` is render-only. This defect is LAYOUT/RESPONSIVE ONLY - do not change marker shapes, the legend semantics, the marker model, or any token/hex.
